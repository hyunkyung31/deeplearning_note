# FFNN 핵심 개념 — 공책용 정리

> TensorFlow / Keras 기반 Feed Forward Neural Network 학습 노트

---

## 1. 전체 뼈대: 딥러닝 한 번 학습시키는 흐름

```
[1] 데이터 준비
      ↓
[2] 모델 설계 (레이어 쌓기)
      ↓
[3] 손실함수 + 최적화기 설정
      ↓
[4] 학습 (예측 → 틀린 정도 계산 → 가중치 수정)
      ↓
[5] 평가 & 예측
```

### 전체 흐름도

```
┌─────────────────────────────────────────────────────────┐
│                    ① 데이터 준비                          │
│  원본 데이터 → 숫자 텐서로 변환 → 정규화 → train/val/test │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│                    ② 모델 (FFNN)                          │
│  입력 → [Dense + activation] × N → 출력(logits)         │
│         Y = activation(X @ W + b)                        │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│              ③ compile (엔진 세팅)                        │
│  optimizer: 가중치를 어떻게 고칠지 (SGD, Adam...)       │
│  loss: 얼마나 틀렸는지 점수 매기는 기준 (MSE, CE...)     │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│                    ④ fit (학습 반복)                      │
│  for epoch:                                              │
│    for batch:                                            │
│      ① 예측(y_pred)                                      │
│      ② loss 계산                                         │
│      ③ 역전파(gradient)                                  │
│      ④ 가중치 업데이트                                    │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│                 ⑤ evaluate / predict                     │
│  test 데이터로 최종 성능 확인 → 새 데이터 예측            │
└─────────────────────────────────────────────────────────┘
```

**한 줄 요약:** 데이터를 숫자로 바꾸고 → 모델이 답을 찍고 → 틀린 만큼을 점수(loss)로 매기고 → 그 점수를 줄이는 방향으로 가중치(W, b)를 고친다.

## 2. 텐서(Tensor)와 변수(Variable)

### 텐서란?

- 딥러닝에서 쓰는 **다차원 숫자 배열**
- NumPy 배열과 비슷하지만, GPU에서 빠르게 계산하려고 TensorFlow/PyTorch가 쓰는 전용 자료형

### 종류 2가지

| 종류 | 코드 | 특징 | 예시 |
|------|------|------|------|
| **상수** | `tf.constant` | 값이 고정, 학습 중 안 바뀜 | 입력 데이터 X, 정답 y |
| **변수** | `tf.Variable` | 값이 바뀜, **학습 대상** | 가중치 W, 편향 b |

### shape(모양) 읽는 법

```
(batch, features)
  │        │
  │        └── 한 샘플의 특징 개수 (MNIST: 784)
  └── 한 번에 넣는 데이터 개수 (64개씩 묶으면 64)
```

**MNIST 예시:**

- 원본: `(60000, 28, 28)` — 6만 장, 28×28 픽셀
- FFNN용: `(60000, 784)` — 28×28을 한 줄로 펼침
- 학습 시 batch: `(64, 784)` — 64장씩 GPU에 전달

### `.numpy()`는 언제?

- **확인용** — "지금 값이 뭐지?" 볼 때만
- 학습 루프 안에서 남용하면 속도가 느려짐

## 3. FFNN 한 층의 수식 — 가장 중요

### 한 층이 하는 일

```
입력 X  ──→  [ ×W + b ]  ──→  [ activation ]  ──→  출력 Y
```

**수식:**

```
Y = activation( X @ W + b )
```

| 기호 | 의미 | shape 예시 |
|------|------|-----------|
| X | 입력 데이터 | (batch, 3) |
| W | 가중치 (학습됨) | (3, 2) ← **(들어오는, 나가는)** |
| b | 편향/절편 (학습됨) | (2,) ← **나가는 개수와 같음** |
| Y | 출력 | (batch, 2) |

### shape 계산 공식 (외워야 함)

```
X: (batch, d_in)
W: (d_in, d_out)    ← Dense(units=d_out)일 때
b: (d_out,)
Y: (batch, d_out)
```

**예시:**

```
입력 X: (100, 784)     ← 100장, 픽셀 784개
Dense(128): W=(784,128), b=(128,)  → 출력 (100, 128)
Dense(64):  W=(128,64),  b=(64,)   → 출력 (100, 64)
Dense(10):  W=(64,10),   b=(10,)   → 출력 (100, 10)
```

### W, b, activation 각각 왜 필요?

| 구성요소 | 역할 | 없으면? |
|----------|------|---------|
| **W (가중치)** | 입력 특징마다 "얼마나 중요한지" 배분 | 아무것도 학습 못 함 |
| **b (편향)** | 출력을 위아래로 밀어줌 (절편) | 0 근처에 갇혀 학습 느려짐 |
| **activation** | 비선형 변환 (ReLU 등) | 층 100개 쌓아도 1층과 같음 |

### ReLU

```
ReLU(x) = max(0, x)
```

- 음수 → 0
- 양수 → 그대로

## 4. 역전파 — 학습의 핵심 4줄

### 비유

```
학생(모델)이 시험 → 채점(loss) → "어디를 고쳐야 하나?"(gradient) → 노트 정리(가중치 수정)
```

### 4줄 코드

```python
# ① loss 계산 (Tape가 중간 과정 기록)
with tf.GradientTape() as tape:
    y_pred = model(x)           # 예측
    loss = loss_fn(y, y_pred)   # 틀린 정도

# ② 기울기(gradient) 계산
grads = tape.gradient(loss, model.trainable_variables)

# ③ 가중치 업데이트
optimizer.apply_gradients(zip(grads, model.trainable_variables))
```

### GradientTape란?

- **계산 과정을 녹화하는 녹음기**
- `tape.gradient()` 호출 시 → "loss를 W에 대해 미분하면?" 자동 계산
- `with` 블록 끝나면 녹음기 삭제 (메모리 절약)

### trainable_variables란?

- 모델 안에서 **학습되는 모든 W, b를 한꺼번에 모아둔 리스트**

## 5. Keras 4단계 — 고수준 API 흐름

```
① 정의 (build)  →  ② compile  →  ③ fit  →  ④ predict
   모델 뼈대          엔진 세팅      학습       예측
```

### Django와 비유

| Django | Keras |
|--------|-------|
| models.py (모델 정의) | Sequential / Dense |
| settings.py (설정) | compile (optimizer, loss) |
| runserver (실행) | fit (학습) |
| view에서 응답 | predict (예측) |

## 6. 손실함수(Loss)

| 문제 유형 | loss | 정답(y) 형태 |
|-----------|------|-------------|
| **회귀** (숫자 예측) | MSE | `[3.5, 7.2, ...]` 실수 |
| **분류** (정수 라벨) | SparseCategoricalCrossentropy | `[0, 2, 5, ...]` 정수 |
| **분류** (원-핫) | CategoricalCrossentropy | `[[1,0,0], [0,0,1]]` |

### from_logits=True

```
모델 출력 ──→ logits (raw score, softmax 전)
                ↓
         loss 함수가 내부에서 softmax + CE를 한 번에 계산
```

| 설정 | 모델 마지막층 | loss 설정 |
|------|-------------|-----------|
| `from_logits=True` | activation **없음** | loss가 softmax 처리 |
| `from_logits=False` | softmax **있음** | loss는 확률값만 받음 |

**주의:** logits인데 softmax를 또 붙이면 → 이중 적용 → loss 틀어짐

## 7. 데이터 3분할 + tf.data 파이프라인

### 데이터 역할

```
전체 데이터
    ├── train (80%)  → 가중치 학습 (fit)
    ├── val   (10%)  → 학습 중 성능 확인, 조기 종료 판단
    └── test  (10%)  → 학습 끝난 후 최종 1회 평가
```

**주의:** test는 **학습이 완전히 끝난 뒤 딱 1번**만 사용.

### tf.data 파이프라인

```
원본 배열
    ↓ from_tensor_slices  (한 줄씩 꺼낼 준비)
    ↓ shuffle             (순서 섞기 — 매 epoch마다)
    ↓ batch               (64개씩 묶기)
    ↓ prefetch            (다음 배치 미리 로딩)
    ↓
  GPU로 전달
```

## 8. @tf.function — 속도 최적화

| 모드 | 특징 | 언제 |
|------|------|------|
| **Eager** | 코드 한 줄 실행 → 바로 결과 | 개발·디버깅 중 |
| **Graph** | 전체를 그래프로 묶어 한 번에 실행 | 학습 확정 후 |

```
1차 호출: 그래프 생성 (느림) ──→ 캐시 저장
2차 호출: 캐시 재사용 (빠름) ──→ 캐시 재사용 (빠름)
```

> 개발할 때는 @tf.function 안 붙이고, 학습 코드 확정 후 train_step에 붙인다.
